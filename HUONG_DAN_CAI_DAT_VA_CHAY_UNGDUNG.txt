I. YÊU CẦU HỆ THỐNG

1. Phần mềm bắt buộc:
   - Java Development Kit (JDK) 17 trở lên
   - Apache Maven 3.6.0 trở lên
   - MySQL Server 8.0 trở lên
   - Node.js 16.0 trở lên
   - npm 8.0 trở lên
   - Flutter SDK 3.9.2 trở lên (cho mobile app)
   - Android Studio (cho phát triển Android)
   - Xcode (cho phát triển iOS - chỉ trên macOS)

2. Cấu hình tối thiểu:
   - RAM: 4GB
   - Dung lượng ổ cứng trống: 2GB
   - Hệ điều hành: Windows 10/11, macOS, hoặc Linux

3. Trình duyệt web được hỗ trợ:
   - Google Chrome phiên bản mới nhất
   - Mozilla Firefox phiên bản mới nhất
   - Microsoft Edge phiên bản mới nhất

II. CÀI ĐẶT MÔI TRƯỜNG PHÁT TRIỂN

A. CÀI ĐẶT JAVA JDK 17

Windows:
1. Tải JDK 17 từ Oracle hoặc OpenJDK
2. Chạy file cài đặt và làm theo hướng dẫn
3. Thiết lập biến môi trường JAVA_HOME:
   - Mở System Properties > Advanced > Environment Variables
   - Tạo biến mới JAVA_HOME với giá trị đường dẫn cài đặt JDK
   - Thêm %JAVA_HOME%\bin vào biến PATH
4. Kiểm tra cài đặt bằng lệnh:
   java -version

macOS/Linux:
1. Sử dụng package manager hoặc tải từ trang chủ
2. Thiết lập JAVA_HOME trong file ~/.bashrc hoặc ~/.zshrc:
   export JAVA_HOME=/path/to/jdk17
   export PATH=$JAVA_HOME/bin:$PATH
3. Tải lại cấu hình: source ~/.bashrc
4. Kiểm tra: java -version

B. CÀI ĐẶT APACHE MAVEN

Windows:
1. Tải Maven từ https://maven.apache.org/download.cgi
2. Giải nén vào thư mục (ví dụ: C:\Program Files\Apache\Maven)
3. Thiết lập biến môi trường:
   - Tạo MAVEN_HOME trỏ đến thư mục Maven
   - Thêm %MAVEN_HOME%\bin vào PATH
4. Kiểm tra: mvn -version

macOS/Linux:
1. Sử dụng package manager (brew install maven hoặc apt-get install maven)
2. Hoặc tải và cấu hình thủ công tương tự Windows
3. Kiểm tra: mvn -version

C. CÀI ĐẶT MYSQL SERVER

1. Tải MySQL Server 8.0 từ https://dev.mysql.com/downloads/mysql/
2. Cài đặt MySQL Server và ghi nhớ:
   - Root password
   - Port (mặc định: 3306)
3. Khởi động MySQL Server
4. Kiểm tra kết nối:
   mysql -u root -p

D. CÀI ĐẶT NODE.JS VÀ NPM

Windows:
1. Tải Node.js LTS từ https://nodejs.org/
2. Chạy file cài đặt (npm sẽ được cài đặt cùng)
3. Kiểm tra:
   node -v
   npm -v

macOS/Linux:
1. Sử dụng package manager hoặc nvm (Node Version Manager)
2. Cài đặt: nvm install --lts
3. Kiểm tra phiên bản như trên

E. CÀI ĐẶT FLUTTER SDK (CHO MOBILE APP)

Windows:
1. Tải Flutter SDK từ https://flutter.dev/docs/get-started/install/windows
2. Giải nén vào thư mục (ví dụ: C:\src\flutter)
3. Thêm flutter\bin vào biến môi trường PATH:
   - System Properties > Environment Variables
   - Chỉnh sửa PATH, thêm đường dẫn C:\src\flutter\bin
4. Kiểm tra cài đặt:
   flutter doctor
5. Cài đặt Android Studio:
   - Tải từ https://developer.android.com/studio
   - Cài đặt Android SDK và Android SDK Command-line Tools
   - Cài đặt Flutter và Dart plugins trong Android Studio
6. Chấp nhận Android licenses:
   flutter doctor --android-licenses

macOS:
1. Tải Flutter SDK từ https://flutter.dev/docs/get-started/install/macos
2. Giải nén vào thư mục (ví dụ: ~/development/flutter)
3. Thêm vào ~/.zshrc hoặc ~/.bashrc:
   export PATH="$PATH:$HOME/development/flutter/bin"
4. Tải lại cấu hình: source ~/.zshrc
5. Kiểm tra: flutter doctor
6. Cài đặt Xcode từ App Store (cho iOS development)
7. Cài đặt CocoaPods: sudo gem install cocoapods
8. Cài đặt Android Studio (cho Android development)
9. Chấp nhận licenses: flutter doctor --android-licenses

Linux:
1. Tải Flutter SDK từ https://flutter.dev/docs/get-started/install/linux
2. Giải nén: tar xf flutter_linux_*.tar.xz
3. Thêm vào ~/.bashrc: export PATH="$PATH:/path/to/flutter/bin"
4. Tải lại: source ~/.bashrc
5. Cài đặt Android Studio
6. Kiểm tra: flutter doctor
7. Chấp nhận licenses: flutter doctor --android-licenses

Lưu ý:
- Chạy flutter doctor để kiểm tra các dependencies còn thiếu
- Đảm bảo tất cả mục trong flutter doctor đều có dấu tích xanh

IV. CÀI ĐẶT VÀ CẤU HÌNH BACKEND

A. TRUY CẬP THỨ MỤC BACKEND

cd "MyFinance-Project\MyFinance Backend"

Lưu ý: Thay đổi đường dẫn phù hợp với vị trí project trên máy của bạn.

B. CẤU HÌNH APPLICATION.PROPERTIES

Mở file: src/main/resources/application.properties

Cấu hình kết nối database:

spring.datasource.url=jdbc:mysql://localhost:3306/myfinance?useSSL=false&serverTimezone=Asia/Ho_Chi_Minh
spring.datasource.username=root
spring.datasource.password=your_mysql_password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

Cấu hình JPA/Hibernate:

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL8Dialect
spring.jpa.properties.hibernate.format_sql=true

Cấu hình JWT:

jwt.secret=your_jwt_secret_key_here_minimum_256_bits
jwt.expiration=86400000

Cấu hình SMTP (cho chức năng email):

spring.mail.host=smtp.mailtrap.io
spring.mail.port=2525
spring.mail.username=your_mailtrap_username
spring.mail.password=your_mailtrap_password
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true

Cấu hình Frontend URL (cho email reset password):

app.frontend.url=http://localhost:3000

Lưu ý quan trọng:
- Thay your_mysql_password bằng mật khẩu MySQL thực tế
- Thay your_jwt_secret_key_here bằng chuỗi ngẫu nhiên dài tối thiểu 256 bit
- Cấu hình SMTP nếu cần sử dụng chức năng gửi email
- Đối với production, sử dụng biến môi trường thay vì hard-code

C. CÀI ĐẶT DEPENDENCIES

mvn clean install

Lệnh này sẽ:
- Tải tất cả dependencies từ Maven Central
- Biên dịch source code
- Chạy unit tests
- Đóng gói ứng dụng thành file JAR

D. CHẠY ỨNG DỤNG BACKEND

Cách 1 - Sử dụng Maven:

mvn spring-boot:run

Cách 2 - Chạy JAR file trực tiếp:

java -jar target/myfinance-backend-0.0.1-SNAPSHOT.jar

Cách 3 - Sử dụng IDE (IntelliJ IDEA, Eclipse):

1. Import project as Maven project
2. Tìm class MyFinanceApplication.java
3. Right-click > Run 'MyFinanceApplication'

E. KIỂM TRA BACKEND HOẠT ĐỘNG

1. Mở trình duyệt và truy cập:
   http://localhost:8080

2. Kiểm tra API endpoints:
   http://localhost:8080/api/auth/check-admin-exists

3. Nếu thấy response JSON, backend đã hoạt động thành công

F. TẠO TÀI KHOẢN ADMIN (LẦN ĐẦU CHẠY)

Sử dụng Postman hoặc curl để tạo admin:

POST http://localhost:8080/api/admin/setup/create-admin
Content-Type: application/x-www-form-urlencoded

Body:
email=admin@myfinance.com
password=admin123
fullName=System Administrator
roleType=ADMIN

Hoặc sử dụng curl:

curl -X POST http://localhost:8080/api/admin/setup/create-admin ^
  -H "Content-Type: application/x-www-form-urlencoded" ^
  -d "email=admin@myfinance.com&password=admin123&fullName=System Administrator&roleType=ADMIN"

VIII. CÁC VẤN ĐỀ THƯỜNG GẶP VÀ CÁCH KHẮC PHỤC

A. BACKEND KHÔNG KHỞI ĐỘNG

Vấn đề 1: Port 8080 đã được sử dụng
Giải pháp:
- Tìm và tắt process đang dùng port 8080
- Hoặc thay đổi port trong application.properties:
  server.port=8081

Vấn đề 2: Không kết nối được MySQL
Giải pháp:
- Kiểm tra MySQL Server đang chạy
- Kiểm tra username/password trong application.properties
- Kiểm tra database myfinance đã được tạo
- Kiểm tra firewall không chặn port 3306

Vấn đề 3: Maven build thất bại
Giải pháp:
- Xóa thư mục ~/.m2/repository và chạy lại mvn clean install
- Kiểm tra kết nối internet
- Kiểm tra Java version: java -version

B. FRONTEND KHÔNG KHỞI ĐỘNG

Vấn đề 1: npm install thất bại
Giải pháp:
- Xóa thư mục node_modules và file package-lock.json
- Chạy lại: npm install
- Hoặc dùng: npm install --legacy-peer-deps

Vấn đề 2: Port 3000 đã được sử dụng
Giải pháp:
- Tìm và tắt process đang dùng port 3000
- Hoặc React sẽ tự động đề xuất port khác (3001, 3002,...)

Vấn đề 3: CORS error khi gọi API
Giải pháp:
- Kiểm tra WebConfig.java trong backend đã cấu hình CORS
- Kiểm tra backend đang chạy tại đúng port 8080

C. VẤN ĐỀ DATABASE

Vấn đề 1: Table không được tạo tự động
Giải pháp:
- Kiểm tra spring.jpa.hibernate.ddl-auto=update
- Hoặc chạy script SQL thủ công từ file migration

Vấn đề 2: Encoding tiếng Việt bị lỗi
Giải pháp:
- Kiểm tra database charset: utf8mb4
- Kiểm tra table collation: utf8mb4_unicode_ci

Vấn đề 3: Mất kết nối database
Giải pháp:
- Thêm vào application.properties:
  spring.datasource.hikari.connection-timeout=30000
  spring.datasource.hikari.maximum-pool-size=10

D. VẤN ĐỀ MOBILE APP

Vấn đề 1: flutter command not found
Giải pháp:
- Kiểm tra Flutter đã được thêm vào PATH
- Chạy: flutter doctor để kiểm tra cài đặt
- Khởi động lại terminal sau khi cài đặt Flutter

Vấn đề 2: No devices found
Giải pháp:
- Kiểm tra emulator/simulator đang chạy
- Kiểm tra USB debugging đã bật (cho thiết bị thật)
- Chạy: flutter devices để xem danh sách devices
- Khởi động lại ADB: adb kill-server && adb start-server

Vấn đề 3: Gradle build failed (Android)
Giải pháp:
- Xóa thư mục build: flutter clean
- Chạy lại: flutter pub get
- Kiểm tra Android SDK đã cài đặt đầy đủ
- Cập nhật Android Studio và Android SDK tools
- Kiểm tra Gradle version trong android/gradle/wrapper/gradle-wrapper.properties

Vấn đề 4: CocoaPods error (iOS)
Giải pháp:
- Cài đặt lại CocoaPods: sudo gem install cocoapods
- Xóa Podfile.lock và Pods: cd ios && rm -rf Pods Podfile.lock
- Chạy: pod install
- Nếu vẫn lỗi: pod repo update

Vấn đề 5: Không kết nối được API
Giải pháp:
- Kiểm tra backend đang chạy tại port 8080
- Android Emulator: Sử dụng 10.0.2.2 thay vì localhost
- iOS Simulator: Sử dụng localhost
- Thiết bị thật: Sử dụng IP của máy (ipconfig hoặc ifconfig)
- Tắt firewall tạm thời để test
- Kiểm tra API URL trong lib/config/api_config.dart

Vấn đề 6: flutter_secure_storage error
Giải pháp:
Android:
- Cập nhật minSdkVersion lên 18 trong android/app/build.gradle.kts
